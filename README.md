# GANDM

GANDM — мультиязычная платформа для поиска грузов, транспорта, складов и
таможенных услуг. Участник использует один кабинет и сам включает нужные
направления работы; тип «физическое/юридическое лицо» хранится отдельно от
набора инструментов.

## Состав проекта

- `cmd/api` — HTTP API на Go и chi;
- `web` — React, TypeScript и Vite;
- `matching` — изолированный Python-сервис группировки грузов без внешних
  зависимостей;
- `migrations` — последовательные миграции PostgreSQL;
- `cmd/seed` — повторяемое заполнение локальных справочников и тестовых данных;
- `scripts/smoke.sh` — сквозная проверка API на отдельной тестовой базе.

## Требования для локальной разработки

- Go 1.25+ (модуль закрепляет актуальный toolchain);
- Node.js 20.19+ или 22.12+;
- PostgreSQL;
- MinIO;
- Python 3.10+;
- `jq` и `curl` для smoke-теста.

## Первый запуск

```bash
cp .env.example .env
# Замените JWT-секреты и параметры локальной БД в .env.
go mod download
cd web && npm ci && cd ..
```

В трёх терминалах:

```bash
python3 matching/main.py --port 8000
./scripts/dev.sh
cd web && npm run dev
```

`scripts/dev.sh` проверяет PostgreSQL и MinIO, применяет миграции и запускает
API на `http://localhost:8080`. Интерфейс Vite доступен на
`http://localhost:5173` и проксирует `/api` на backend.

После первого старта можно добавить локальные демонстрационные данные:

```bash
go run ./cmd/seed
```

## Проверки перед передачей изменений

```bash
go test ./cmd/... ./internal/...
go test -race ./cmd/... ./internal/...
go vet ./cmd/... ./internal/...
go build ./cmd/... ./internal/...
python3 -m unittest discover -s matching
cd web
npm test
npm run lint
npm run build
npm audit
```

Smoke-тест изменяет данные и поэтому должен запускаться только на отдельной
локальной тестовой базе:

```bash
DATABASE_DSN='postgres://.../gandm_smoke?sslmode=disable' ./scripts/smoke.sh
```

## Важные правила модели доступа

- `legal_form` отвечает только за юридический статус и перечень обязательных
  документов;
- возможности участника определяются активными инструментами, а не старым
  полем роли;
- после изменения имени или юридического статуса требуется повторная
  верификация;
- публикация собственного груза бесплатна; платные и внешние функции должны
  проверяться на backend независимо от интерфейса;
- координаты в API и PostgreSQL всегда WGS-84.

Настройки перечислены в `.env.example`. Реальные секреты и локальные `.env`
файлы в репозиторий не добавляются.
