# Фронтенд платформы (Этапы 1–2)

React + TypeScript + Vite. Три кабинета: админка, кабинет клиента, кабинет участника.
Публичной регистрации в UI пока нет — участники создаются через POST /api/register или seed.

## Запуск

```
cd web
npm install
npm run dev
```

Откроется на **http://localhost:5173**.

Перед запуском фронта должен быть поднят backend (`../scripts/dev.sh` из корня репозитория —
Postgres, MinIO, миграции, `go run ./cmd/api` на `:8080`).

## Входы и тестовые учётки (из `go run ./cmd/seed`)

- `/admin/login` — админка: `admin@platform.local` / `Admin12345!`
- `/login` — участники (пароль у всех `Test12345!`):
  - `client.active@example.com` — кабинет клиента (заявки на груз, анонимные предложения)
  - `warehouse.active@example.com` — кабинет участника (направления, доступные грузы,
    предложения, уведомления; инструменты и маршруты уже назначены seed'ом)

Кабинет выбирается по `participant_type` из ответа логина: `client` → кабинет клиента,
остальные — кабинет участника. Это только навигация: доступ к действиям по-прежнему
проверяется backend'ом через инструменты (принцип Блока А).

## CORS / прокси

Фронтенд не ходит напрямую на `localhost:8080` — в `vite.config.ts` настроен dev-прокси:
всё, что начинается с `/api`, Vite сам перенаправляет на backend. Для браузера это
same-origin запрос, поэтому для локальной разработки CORS на backend не нужен и мы его
не трогали.

Если когда-то фронтенд и backend будут жить на разных доменах без прокси между ними
(например, в проде) — тогда backend'у понадобится CORS. На этот случай (ничего сейчас
делать не нужно) для chi можно добавить `github.com/go-chi/cors`:

```go
import "github.com/go-chi/cors"

r.Use(cors.Handler(cors.Options{
    AllowedOrigins: []string{"https://ваш-фронтенд-домен"},
    AllowedMethods: []string{"GET", "POST", "PATCH", "DELETE", "OPTIONS"},
    AllowedHeaders: []string{"Content-Type", "Authorization"},
}))
```

## Переменные окружения

`.env.example` → `.env`. По умолчанию `VITE_API_BASE_URL=/api` (использует прокси выше).
Меняйте, только если фронтенд должен ходить на backend напрямую, минуя прокси.

## Структура

```
src/
  api/          — fetch-клиент, типы DTO, типизированные вызовы под каждый /admin/* endpoint
  auth/         — контекст авторизации (токен в памяти + localStorage)
  i18n/         — словарь строк интерфейса (сейчас только ru, структура готова под en/zh)
  components/   — layout (сайдбар + рабочая зона) и переиспользуемые UI-примитивы
  hooks/        — useAsync — единый паттерн loading/error/data для запросов
  pages/        — 5 экранов: Login, Dashboard, Verification, Users, Tools
  styles/       — global.css
```

## Карты и координаты

- Точки отправления/назначения выбираются на карте (формы заявки, направлений,
  карточка пользователя в админке). Переключатель «Казахстан (OSM)» / «Китай (Amap)».
- **Казахстан/прочее**: Leaflet + OpenStreetMap, поиск через публичный Nominatim
  (дебаунс 700 мс, минимум 3 символа — чтобы не упереться в лимит запросов).
- **Китай**: Amap (Gaode). Нужен ключ: скопируйте `web/.env.example` в `web/.env`
  и заполните `VITE_AMAP_KEY` (префикс VITE_ обязателен — ключ используется в браузере).
  Без ключа вкладка «Китай» показывает подсказку, а не молчаливо ломается.
- **Система координат**: в API и БД уходит только WGS-84. Amap отдаёт GCJ-02 —
  конверсия происходит в `src/utils/gcj02.ts` внутри `GeoPointField`, до отправки.
  Поле `source` (amap/osm) — метаданные для отладки, а не флаг конверсии.
- После обновления зависимостей выполните `npm install` заново (добавлен leaflet).

## Уведомления

- Бейдж непрочитанных в кабинете участника обновляет **единственный на всё приложение
  поллер** (`src/notifications/poller.ts`): один интервал в 30 сек (`NOTIFICATIONS_POLL_MS`),
  защита от параллельных запросов, старт с первым подписчиком и остановка с последним —
  сколько бы компонентов ни монтировалось/размонтировалось, опрос не размножается.
- Открытие раздела «Уведомления» вызывает `POST /api/notifications/read` (после загрузки
  списка, чтобы точки непрочитанного были видны в этом заходе) и сразу обнуляет бейдж.

## Известные ограничения

- Лента «последних действий» на дашборде берётся из `GET /api/admin/audit-log`
  (реальное время действий админа) — старая приблизительная логика удалена.
